# Cheatsheet — The Minto Pyramid Principle

## Decide: deductive or inductive?
| If the support points… | It's… | Summarize by… |
|---|---|---|
| comment on each other in sequence → "therefore" | **Deductive** | paraphrasing the "therefore" (rest on final point) |
| are the same kind of thing, named by one plural noun | **Inductive** | stating what's alike (the inductive leap) |
Rule: one or the other per grouping, never both.

## Decide: which order?
- Formed by **dividing a whole** → **structural order** (must be MECE).
- Formed by **cause-and-effect / steps** → **time order**.
- Formed by **classifying/ranking** → **degree order** (importance).
- Can't say why point 2 comes second → you haven't finished thinking; re-sort.

## Decide: which introduction pattern?
| Reader's question | Pattern |
|---|---|
| What/How should we do it? | Giving Direction (Key Line = steps) |
| Should we spend money? | Seeking Approval (4 standard reasons) |
| How do we do it? | Explaining How-to (present vs. recommended process) |
| Which one? | Choosing Alternatives (alternatives → Complication) |

## Problem → question (where does the reader stand?)
- Don't know how to get R1→R2 → "What should we do / How?"
- Have a proposed solution, unsure → "Is it the right solution?"
- Know the solution, can't implement → "How do we implement it?"
- Tried it, it failed → "What should we do?" (trace the layers first)
- Several candidates → "Which alternative?" (they go in the Complication)

## Hard numbers & defaults
- Chained **deductive** argument: never more than **~4** points.
- **Inductive** group: never more than **~5** points; mind holds only ~7 at once.
- **30-second rule**: reader should get Introduction + Main Point + Key Line almost immediately.
- **R2** must be specific/quantifiable — or you can't choose a solution.
- Introduction = **remind, not inform** → zero exhibits/proof in it.
- SCQA convert: read the problem framework left-to-right-and-down; **last thing the reader knows = the Complication**.

## Tells & smells (you're in trouble if…)
- Your summary point is a **count** — "the company has five problems," "three objectives." → Blank assertion; state the effect/implication instead.
- You wrote an **"Our Assumptions" / "Background"** section before the points. → You're answering unraised questions.
- Your **ToC reads Findings / Conclusions / Recommendations**. → The boxes hold labels, not ideas.
- You **can't name a group with one plural noun**. → Not a valid grouping; the ideas don't belong together yet.
- You're **listing alternatives to knock them down** ("A is bad, B is bad, therefore C"). → You choose C because it solves the problem, not because A/B are bad.
- You're **collecting data "to be sure."** → Form hypotheses first; gather only to answer yes/no.
- A **heading stands alone** at its level, or you have only one subsection. → Never one of any element.

## The whole method in one line
Decide the single point (top-down, via SCQA) → group its support MECE and order it (time/structural/degree) → summarize each group with a real insight (not a count) → lay it out so the eye sees the pyramid → put it in words by copying the image you saw.
