## Core Frameworks & Mental Models

**The Pyramid Principle (the whole book in one idea).** The mind sorts information into pyramids to comprehend it, so structure every document as a pyramid of ideas under a **single Main Point**, delivered **top-down**. Three rules make it a pyramid, and you test them *before* writing: (1) any point **summarizes** the ideas below it; (2) ideas in a group are **the same kind** (nameable by one plural noun); (3) ideas in a group are **logically ordered**. A broken rule signals flawed thinking, not just bad prose. (Minto's apex term is the **Main Point**; secondary sources call it the "governing thought.")

**Three substructures for discovering the pyramid** (Ch 2): the **vertical** question/answer dialogue (every idea is *a statement that raises a question*; you answer it on the line below, down to where the reader has no more questions); the **horizontal** logic (each group is **deductive OR inductive**, never both); and the **introductory flow** (SCQA).

**SCQA introduction** (Ch 2, 4): open in story form — **Situation** (what the reader accepts) → **Complication** (what changed) → **Question** (what they now ask) → **Answer** (your Main Point). Introductions **remind, they don't inform** — no proof or exhibits. There is always exactly one beginning Question.

**Build top-down** (Ch 3) — the box procedure: draw the top box → Subject → Question → Answer → Situation → Complication ("Yes, I know that, *so what?*") → recheck that the Complication triggers the Question → answer the New Question on the **Key Line**. Use bottom-up (list → relate → conclude) only when the apex is unknown.

**Deduction vs. induction** (Ch 5): **deductive** points derive from each other in steps to a "therefore" (summarize resting on the final point; never chain beyond ~4); **inductive** points are same-kind members of a class named by a plural noun (summarize by the inference; never beyond ~5). Deduction is how we think but a ponderous way to write — prefer induction for presentation.

**Impose logical order** (Ch 6): only three valid orders, each tied to how the group was formed — divide a whole → **structural order** (must be **MECE** = Mutually Exclusive, Collectively Exhaustive, Minto's coinage); trace cause-effect → **time order**; classify → **degree order** (importance). If you can't say why point 2 comes second, keep thinking.

**Summarize with a real point — kill the "blank assertion"** (Ch 7): never top a group with a count ("the company has five problems"). Reword each point as a full sentence, find what's alike, and state it — for **actions**, their combined **effect**; for **situations**, their **implication** (the inductive leap). A real summary inspires further thinking.

**Define the problem as a gap** (Ch 8): a problem is the gap between the **Undesired Result (R1)** and the **Desired Result (R2)**. Lay out **Situation** (Opening Scene + Disturbing Event) → R1 → R2; state R2 specifically/quantifiably. This converts left-to-right-and-down into the SCQA introduction (last thing known = Complication). A well-defined problem has **no alternative solutions**.

**Structure the analysis — hypotheses first** (Ch 9): don't "go after everything." Use a **diagnostic framework** (show physical structure / trace cause-effect trees / MECE classification of causes) to derive likely causes, then gather data only to answer **yes/no**. Diagnostic frameworks find causes; **logic trees** generate recommendations — keep them distinct.

**Reflect the pyramid in presentation** (Ch 10–12): on the **page**, use hierarchical headings so the eye sees the logic (Introduction + Main Point + Key Line in ~30 seconds); on the **screen**, text slides carry major points, exhibit slides carry data, all storyboarded; in **prose**, "see" the idea as an image, then copy the image into words.
