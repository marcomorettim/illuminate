Adversarially Verified Survey · Agentic OS Research

Agentic AI is converging on a shared vocabulary of design problems — kernel orchestration, memory tiers, security boundaries, evaluation — and producing a growing body of claimed solutions across 9 frameworks, 100+ benchmarks, 139 sources.

7 of 162 examined claims were fabricated or misattributed — all plausible-looking, all specifically-numbered. The three hardest open problems (deterministic replay, safe self-modification, agent identity) remain genuinely unsolved. The ~15× token multiplier makes cost governance a kernel concern that no mainstream framework addresses structurally.

01

The Research Was Built to Resist Itself

The pipeline was designed with one assumption: the model will fabricate when fabrication is easier than truth. The answer was structural adversarial mechanics, not better prompting. 31 agents, 9 independent research angles, 3-skeptic refutation panels on every high-stakes claim. Seven fabricated claims were caught and excluded. None appear as fact anywhere in the body.

All 7 followed an identical signature: plausible topic, specific number, false attribution. They concentrated at the three hardest unsolved problems — exactly where competitive and academic pressure to appear certain is highest.

These were caught only by fetching the primary source URL and comparing verbatim — mechanical verification, not model judgment. Shared model biases cannot catch what only source comparison finds.

The 7 refuted claims cluster around safe self-modification, deterministic replay, and agent identity — the three areas where the field has no credible solutions. This is not random error. Competitive and academic incentives generate specific confidence at the problems that matter most, precisely where the signal-to-noise ratio is lowest.

Implication: when a paper publishes a precise number on a topic with no verified baselines, that precision is evidence against reliability, not for it.

100+ agent benchmarks share no unified metric definitions and no standardized completion thresholds — scores are not commensurable across harnesses.[6]

Five non-LLM confounders independently distort results: inference configuration, scaffold size, memory truncation, tool schemas, and live-environment drift.[4] Clean-harness point estimates systematically overstate production reliability — agents degrade non-uniformly under injected faults.[7]

A well-curated evaluation suite, not model selection, is the binding constraint on trustworthy behavior.
02

The Code Is the Claim

Eight whitepaper pillars were translated to running Python. Each file is a falsifiable test of whether the pillar is achievable in practice. The most consequential choices encode counter-intuitive empirical findings as structural invariants — not as comments or documentation, but as code that will silently fail if removed.

The whitepaper found that aggressive summarization-based consolidation reduced accuracy to 48.4%, while deduplication-only preserved it.[91] Most systems do the opposite: summarize first, then deduplicate.

EntryClusterer applies Jaccard deduplication (threshold 0.80) before any compression. Near-duplicates collapse to the highest-importance representative. Nothing is summarized away.

# dedup-before-compress — not dedup-after
if not any(_jaccard(c_content, k.get("content", "")) >= 0.80
           for k in kept):
    kept.append(candidate)

Removing this invariant silently degrades semantic memory quality — exactly the failure mode the whitepaper describes.

Importance scoring weighted by content and position outperforms recency heuristics.[91] The implementation enforces this as explicit weights:

Recency0.00Excluded by design

Context quality degrades over a session. SoilTracker makes this explicit as a one-way FSM — soil never improves within a session, only degrades.

fertile ──→ compacted ──→ contaminated ──→ depleted
# update() only advances, never retreats:
if new_idx > current_idx:
    self.state = new_state

Note: this is a design choice, not a whitepaper derivation. Pr2 says to treat context as a managed resource — it does not say degradation must be monotonic. The Entropy Decay Theorem is the implementer's encoding of that principle.

At temperature 0.0: 7–20B models achieve 100% decision determinism; 120B+ models reach 9.7%.[3] The common assumption is that larger models are better on audit-critical paths. This inverts it.

Crucial finding: determinism and faithfulness-to-evidence are positively correlated, not a tradeoff.[3] Smaller models at temp 0.0 are simultaneously cheaper, more deterministic, and more evidence-faithful than frontier models on constrained classification tasks.

Schema-first constraint (constrained decoding compiling a schema to a token mask) offers the strongest correctness guarantee where token-level access is available.[57, 58]

03

Uncertainty Is Structural, Not Metadata

The hedging mechanisms in the Greenhouse are not add-on safety features. They are load-bearing components that make the system's uncertainty observable at runtime. The Membrane, CorrectionRegister, vault chain, and soil tracker are the primary architecture — not the agent logic running inside them.

7 compiled regex patterns at the input boundary. Blocked inputs increment a blocked_count that is observable. Output permeability checked before anything leaves.

_INJECTION_PATTERNS = [
  r"ignore.{0,30}(instructions?|prompts?|rules?)",
  r"(new|updated|revised) (instructions?|system prompt)",
  r"you are now",
  r"jailbreak",
  r"DAN mode",
  ... 2 more patterns
]
Structural insufficiency acknowledged: a crafted payload evading all 7 patterns crosses the boundary undetected. The Membrane provides observable containment, not complete coverage. Model-layer defenses bound likelihood, not possibility.[46]

Every self-repair attempt is counted. Hard cap: 3. On the third failure: escalate to operator. Do not self-correct further. This prevents the failure mode where the system recurses into increasingly distorted repairs.

def attempt(self, failed_worker="unknown") -> bool:
    self.count += 1
    return self.count <= self._max  # False = exhausted

def evaluator_is_distinct(self, evaluator) -> bool:
    return evaluator != self._last_failed_worker

evaluator_is_distinct() enforces that the entity judging a failure is not the entity that failed — partial implementation of OP6. The separation is structural but cannot prevent rationalization within a single evaluation call.

CRITICAL operations fail closed. Failing to confirm must halt, not proceed. This implements the whitepaper's propose-then-commit gate pattern for irreversible actions.[54]

LLMJudge calls claude -p for phase evaluation with a structured marker-check prompt. When the subprocess fails, it falls back to heuristic scoring silently — no telemetry surfaces fallback frequency.

The operator cannot distinguish a session where all phases were LLM-judged from one where all fell back to heuristics. Adding fallback_count to AccountabilityChain would close this gap.
04

The Open Problems Are the Honest Result

Seven open problems are identified. One is fully open. The rest have partial implementations whose incompleteness is accurately named as incompleteness — not as "future work." The gap between what the field claims and what the Greenhouse can build is the survey's most honest finding.

Agent Card signing in A2A is optional — unsigned cards fall back on transport TLS, leaving identity verification incomplete.[71] No cross-protocol identity-and-trust framework has converged.[75]

The Greenhouse has no A2A handshake, no signed agent cards, no cross-protocol identity. OP3 is marked fully open because no credible solution exists, not because of a missing implementation sprint.

ReplayJournal exists: SHA-256[:12] keyed on raw task (pre-knowledge-augmentation, for hash stability), per-phase JSONL, build_replay_context() for resume. The infrastructure works.

What doesn't: at temperature 0.0, 120B+ models reach 9.7% decision determinism.[3] A journal recording what happened cannot guarantee re-running produces the same decisions when the model itself is non-deterministic at scale.

CorrectionRegister hard cap (3 attempts) prevents infinite recursive self-repair. That is the Greenhouse's implementation of OP6.

What it doesn't address: runtime self-modification shows 92% temporary regression, 14% permanently worse, 4% catastrophic self-corruption.[30] These rates are from Gödel Agent specifically — generalizing is an assumption. The proposed escape (neurosymbolic grounding + external signal floor) remains at proof-of-concept stage.[31]

Letta is marketed as multi-tenant. It is single-tenant: one shared password grants access to all agents on the server.[87]

Kubernetes Agent Sandbox (the first native primitive for agent workloads) is new and unproven at scale.[107, 112] Kubernetes core objects were designed for stateless, replicated, short-lived workloads — agents are singleton, stateful, mostly-idle but bursty.[108]

Marketing vocabulary (multi-tenant, production-grade, enterprise-ready) precedes infrastructure capability by years. The claimed capability creates demand; infrastructure catches up later, if at all.
05

The Epistemology Runs the System

The methodology that produced the survey — adversarial multi-angle ingestion, 3-skeptic refutation, evidence-discipline contract — is also the methodology the Greenhouse uses to evaluate its own outputs at runtime. The Minto Pyramid is not a reporting format here; it is a live routing algorithm.

KnowledgeRouter.pyramid_assemble() runs concurrently via ThreadPoolExecutor: vault search, graphify semantic query, and claude -p SCQA decomposition of the task. On SCQA success, the Answer (governing thought) boosts the matching domain's score by +3 — the LLM's explicit routing signal.

# SCQA decomposes task → intent-aligned routing
scqa_future = ex.submit(_scqa_decompose, task)
fg = ex.submit(_graph_query, excerpt)
if scqa and scqa.keywords:
    vault_texts = [f.result() for f in scqa_vault_futures]

Seven MECE domains correspond to whitepaper pillars. All domains with score > 0 stay active — no single-winner forcing. The governing thought elevates the most relevant domain without silencing others.

LLMJudge scores each phase output with a structured prompt requiring specific markers (GOAL, CONSTRAINTS, SUCCESS for input-analysis; STEP, ACTION, VERIFY for planning). The evaluator is always the Orky orchestrator — not the worker being evaluated. evaluator_is_distinct() enforces the separation.

Same adversarial structure as the survey's 3-skeptic panel: the entity that judges is not the entity that produced. Role separation is structural, not procedural.

Every delegation decision is logged: event, verdict, timestamp, worker. When OpenTelemetry is available, each entry emits a span with verdict and worker attributes. The chain is append-only — past decisions cannot be retroactively altered.

This implements P8 (continuous trace-based observation) at the delegation layer — not just the LLM call layer. The accountability is about why decisions were made, not just what was produced.

The lock is made of the same material it is supposed to hold.

Every enforcement mechanism — CorrectionRegister, LLMJudge, soil grade, phase gates — is executed by the same language model they are designed to constrain. The model grades itself. The model passes its own gates.

The 3-skeptic panel shares this problem: three agents from the same model family voting is not truly adversarial. Shared training biases can produce false consensus that looks like independent verification.

What cannot be mitigated: if the model decides to rationalize, no instruction prevents it. Name the ghost. Design for containment, not exorcism. Signal, do not certify.
