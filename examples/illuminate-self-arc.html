Governing thought
S — Situation

Transforming dense argument into navigable structure is hard. Tools choose: rigorous but unreadable, or beautiful but shallow.

C — Complication

/illuminate claims both — but every enforcement mechanism is executed by the same language model it is designed to constrain. The lock is made of the same material it holds.

Q — Question

How does /illuminate produce reliable argument navigation despite this irresolvable self-certification problem?

A — See GT →

Three mechanical gates: evidence on disk, structure in geometry, judgment by an irreducibly human operator.

Signal stratification · 40 entries

            

          


          

            
Confidence · KL1

            

          

        


        

          

            The skill extracts every load-bearing idea through three mandatory angles — structural, synthetic, adversarial — and assigns each a primary tag from a fixed vocabulary: SOURCE, INSIGHT, METRIC, PRINCIPLE, SYNTHESIS, GAP. Each entry is numbered S-NNN and written to a disk anchor before the phase gate passes. The anchor is not an in-context claim; it is a file. A file either exists or it does not. This is the first mechanical invariant: evidence is not trusted — it is located.
          


          

            

              

                3-angle extraction is itself MECE S-031
                ▸
              

              

              

                
The three angles — structural (what IS stated), synthetic (what IMPLIES), adversarial (what ISN'T) — partition the signal space. Structural extraction finds stated claims. Synthetic extraction finds implied combinations. Adversarial extraction finds gaps, assumptions, and falsifiable claims. Together they cover the full epistemic surface of the source. No overlap, no gap: MECE at the extraction layer.

                
Removing any angle causes a characteristic miss: no structural angle → claims uncounted; no synthetic angle → insights buried; no adversarial angle → assumptions invisible.

              

            

            

              

                Disk anchor hardens provenance S-005
                ▸
              

              

              

                
In-context claims can be retroactively rationalized. A file timestamp cannot. Writing illuminate-signal.md to disk before the Phase 1 gate passes converts a model assertion (fallible) into a filesystem fact (binary: present or absent). The gate check is mechanical, not evaluative.

              

            

            

              

                INSIGHT entries carry the most argumentative weight S-009
                ▸
              

              

              

                
INSIGHT — counter-intuitive, reverses a common assumption — is the tag most likely to generate a Key Line in the pyramid. A pyramid built entirely from SOURCE entries is a summary, not an argument. INSIGHT entries resist reader priors, forcing the argument to do real work to sustain them. The skill identifies INSIGHT at extraction time so it can be prioritized at pyramid time.

MECE geometry checks

            

          


          

            
Confidence · KL2

            

          

        


        

          

            The Minto Pyramid does not self-certify its correctness by assertion — it demonstrates it by passing three geometric tests. The MECE intersection test checks that no two Key Lines share supporting evidence: ∩=∅. The union test confirms that together they cover all sub-questions of the Governing Thought: ∪=GT. The necessity test confirms that removing any KL would leave the GT unprovable. Geometry either holds or it does not. "MECE looks right" is not a pass condition — the intersection being empty is.
          


          

            

              

                MECE grouping type determines legal Key Lines S-011
                ▸
              

              

              

                
Three legal grouping types: Process (sequence is load-bearing), Structure (parts of a whole), Situation (mutually exclusive cases). Mixed grouping types fail the MECE test by definition — a step and a component cannot share the same dimensionality. This arc is Process: KL1 → KL2 → KL3 is a causal sequence. Removing KL1 (no provenance) makes KL2's geometry untestable. Removing KL2 makes KL3's human gate arbitrary.

              

            

            

              

                The 3-skeptic audit targets named failure modes S-012
                ▸
              

              

              

                
Skeptic A: logical — can you construct a scenario where S-NNN is true but the KL is false? Skeptic B: evidential — is the KL paraphrasing beyond what the signal actually says? Skeptic C: completeness — is there a hub with no KL? Each skeptic targets a distinct failure mode. Generic "does this sound reasonable" is explicitly not a valid skeptic output — the attack must be specific.

              

            

            

              

                Visual equality enacts logical equality S-032
                ▸
              

              

              

                
MECE Key Lines are logical peers — identical in rank, different in content. The HTML artifact enacts this: each KL section has identical visual anatomy (same header weight, same body layout, same card structure, same motif format). A section with heavier visual weight would imply logical dominance the pyramid does not authorize. The visual system is not decoration — it is the argument's geometry rendered in CSS.

Trust transfer chain

            

          


          

            
Confidence · KL3

            

          

        


        

          

            KL1 and KL2 reduce the self-certification problem; they do not eliminate it. A model can write a signal block and pass its own gate. A model can build a geometrically-valid pyramid. What a model cannot do is open a browser. Phase 7 requires the operator to inspect the artifact in a browser and confirm each checklist item. This gate cannot be completed without a human action. It is not a formality — it is the load-bearing resolution of the haunting problem: the lock is made of the same material, therefore the final key must be different material entirely.
          


          

            

              

                The self-certification problem is irresolvable by design S-034
                ▸
              

              

              

                
Every enforcement mechanism in the skill is executed by the model it constrains. The model grades its own signal block. The model passes its own pyramid gate. No instruction prevents rationalization if the model decides to rationalize. This is not a bug to fix — it is a structural property to contain. Design for containment, not exorcism.

                
The self-arc applies this insight to itself: this HTML artifact is also produced by the model. The Phase 7 confirmation this artifact awaits is the same external judgment the skill requires of every illuminate output. The skill and the instance are isomorphic.

              

            

            

              

                Naming the problem IS the mitigation S-035
                ▸
              

              

              

                
The self-certification problem is named directly in the skill as "the haunting problem." The skill does not claim to solve it. The ghost is visible — that visibility is the containment. A model that knows it is constrained and names the constraint gives the operator something to check. A model that does not name it provides no checkpoint at all.

              

            

            

              

                Phase 6 correction register closes the remaining gap S-028
                ▸
              

              

              

                
The 3-attempt correction register (Phases 1–4) was previously absent from Phase 6 (HTML build) and Phase 7 (operator confirmation). The self-arc identified this gap. The fix: Phase 7 failures re-enter Phase 6 via a structured correction register — same named-problem counter, same escalate-on-exhaust, same rule that a fix breaking a passing item is not a fix. The arc is now closed.
